# 公司系统整体规划（2025-12-12）

## 目标
- 一次性实现公司系统的后台管理、前台体验与后端支持，确保在上线后即可支撑玩家注册、审核与管理使用。
- 打造可复用的流程与配置机制，后续业务可以复用流程模板、角色身份、行业定义等资产。
- 明确权限边界：仅管理员可在后台编辑所有企业与个体工商户数据；持有者/法人身份的玩家可在前台 `/company/dashboard` 管理自身公司。

## 核心模块
1. **后端**：统一模型（公司/个体户、行业、法人/持有者/持股人角色、制度、流程实例与记录）+ 接口，上游流程与审批按可复用引擎运行，并提供行业与公司类型配置能力。
2. **前端**：后台管理页用于管理员集中审核与编辑；前台 `/company` 与 `/company/dashboard` 页面分别面向访客推荐与授权用户操作，并展示流程进度与制度内容。
3. **社区与展示**：前台不限登录用户可浏览推荐公司（最近注册/活跃），每条记录展示类型、状态、法人与行业标签。
4. **流程引擎**：定义模板、节点、角色、动作、通知、行业触发规则，支持日后引入更多审批业务。

## 拓展细节
- 公司类型、行业、制度、持有者身份等都可通过配置引入，后续可用“业务配置平台”统一管理；公司类型模板可关联默认流程节点与所需文件。
- 行业设计需要预设分类规则（如科技/制造/服务/金融/文化等）并允许新增，便于搜索与推荐。
- 行业与制度更新、公司状态变化可触发推荐权重调整或通知。
