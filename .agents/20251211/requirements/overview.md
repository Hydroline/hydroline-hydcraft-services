# 公司系统整体规划（2025-12-11）

## 目标
- 构建一个包含后台管理、前台展示与后端接口的“公司系统”基础模块。
- 作为后续其他业务的基础设施，确保审核流程具备高可复用性与高扩展性。
- 兼容玩家注册公司、管理员审批及工商系统正式录入的场景，预留个体工商户等扩展维度。

## 范围
1. 后台：新增公司管理页面，提供审核操作与流程状态可视化。
2. 前台：在 usershell 左侧导航与路由中增加 `/company` 入口，延续现有风格。
3. 后端：设计公司/个体工商户模型、注册与查询接口，并提供流程状态与审批动作。
4. 流程系统：抽象一个可自定义的流程引擎，用于承载“公司审核”及未来其他业务。

## 依赖与原则
- 遵循当前项目已有的前后端样式与认证体系。
- 所有新接口尽可能复用现有服务或工具（如权限、附件、通知等）。
- 流程节点与状态应支持组合：多个角色、多层级审批、补充材料、驳回重审。

## 下一步建议
1. 明确核心数据模型与流程阶段（公司、个体工商户、审核记录、流程状态）。
2. 设计流程引擎的抽象接口，定义状态转换、审批动作与权限校验。
3. 制定后台与前台界面清单，明确哪些组件/页面需要配合流程状态展示。
4. 确定后端 API 列表，并预留通知、日志等扩展能力。
