# 前端需求拆解

## 后台管理页（风格参考现有管理面板）
- 页面路径：例如 `/admin/company-management`，导航/路由与现有后台一致。
- 功能模块：
  1. 公司申请列表（支持按状态/类型、提交时间排序）。
  2. 筛选与搜索栏（按申请人、公司名称、统一社会信用代码）。
  3. 流程状态列，展示当前流程节点与节点状态。
  4. 详情弹窗或侧边栏：展示申请资料、关联流程记录、附件、历史审核意见。
  5. 审核表单：管理员可选择动作（通过/驳回/补件），填写说明并上传附件（如补充证明）。
  6. 流程视图（可复用组件）：可视化呈现已完成节点、当前节点、待办人员。
- 数据来源：调用后端分页 API + 审批动作接口。

## 前台 `/company` 页面
- 路由：在 usershell 导航左侧与 `router` 配置中添加 `/company` 入口。
- 页面内容：
  1. 展示玩家申请的公司状态（名称、类型、审核流程进度、最近审核结果）。
  2. 场景引导：若未注册提示「立即创建公司」按钮，跳转至申请表单。
  3. 申请表单组件（可复用到独立 stepper）：填写公司名称、类型、统一社会信用代码/身份证号、联系人信息、营业地址、基本证明材料。
  4. 与流程系统联动，展示当前节点与预期动作（如需补件显示说明与附件上传按钮）。
  5. 若申请被驳回或待补件，提供重新提交的入口与修改说明。
- 设计风格应与当前前台一致，保持 `usershell` 布局统一，使用现有 UI 组件库。

## 体验细节
- 申请过程中应告知预计审核时长，若流程状态支持「超时提醒」，前端可展示倒计时。
- 审核记录/审批意见列表，按时间倒序展示，支持展开查看详细说明与附件。
- 可考虑复用现有表单校验与文件上传组件，避免重复造轮子。
